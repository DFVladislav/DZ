# «Основные команды оболочки операционной системы GNU/Linux — работа с файлами и каталогами»

**Цель занятия** — ознакомиться с основными командами операционной системы GNU/Linux, командами оболочки для работы с файлами и каталогами.

Изучаются следующие **основные вопросы**:
1. Навигация по файловой системе и просмотр списка файлов и каталогов;
2. Создание и удаление файлов и каталогов в файловой системе;
3. Копирование и перемещение файлов и каталогов в файловой системе;
4. Копирование и перемещение файлов и каталогов в файловой системе;
5. Использование шаблонов при работе с файлами и каталогами;
6. Поиск файлов и каталогов;
7. Создание и использование ссылок на файлы и каталоги;
8. Оценка занятого и свободного пространства.

## Задание

Практическое занятие проводится на компьютерах с установленной свободно-распространяемой и бесплатной операционной системой **Canonical GNU/Ubuntu 12.04 LTS** или **Debian 7** с окружением рабочего стола **GNOME**.

### Подготовка оборудования и работе

Попросите преподавателя включить компьютер и авторизуйтесь в системе с помощью полученных учетных данных (логина и пароля).

### 1. Навигация по каталогам файловой системы и просмотр списка файлов и каталогов

Откройте эмулятор терминала (сочетание **Ctrl+Alt+T** или через меню **Applications → Accessories → Terminal**).

*   Введите `pwd`, чтобы увидеть, в каком каталоге вы находитесь. Запишите путь в отчёт (оформляйте его в **LibreOffice Writer**, который находится в **Applications → Office → LibreOffice Writer**).
*   Наберите `ls` и посмотрите, какие объекты есть в текущей папке. Сохраните результат.
*   Выполните `ls -a`, чтобы отобразить скрытые файлы. Зафиксируйте вывод.
*   Используйте `ls -l` для вывода подробного списка объектов. Сохраните вывод в отчёт.
*   Перейдите в каталог `/bin` с помощью `cd /bin` и выполните `ls`, чтобы увидеть его содержимое.
*   Введите `ls -l`, чтобы получить детализированный список в `/bin`. Сохраните результат.
*   Перейдите в каталог `/sbin/` и выполните `ls -l`, затем сохраните вывод.
*   Вернитесь в домашний каталог командой `cd ~` (без дополнительных символов). Сохраните результат.
*   Перейдите в предыдущий каталог с помощью `cd -`. Запишите вывод.

### 2. Создание и удаление файлов и каталогов в файловой системе

Действия настоящего пункта выполняются внутри каталога `/tmp`.

*   Переместитесь в `/tmp` с помощью команды `cd /tmp`. Сохраните результат.

**Создание и удаление каталогов:**

*   Создайте каталог `newdir1` командой `mkdir newdir1`. Проверьте его наличие с помощью `ls -d /tmp/newdir1`. Сохраните вывод.
*   Удалите `newdir1` командой `rmdir newdir1`. Проверьте удаление через `ls -d /tmp/newdir1`. Сохраните вывод.
*   Создайте вложенные каталоги `newdir1/newdir2` с помощью `mkdir -p newdir1/newdir2`. Проверьте их существование через `ls -d /tmp/newdir1/newdir2`.
*   Удалите вложенные каталоги командой `rmdir -p newdir1/newdir2`. Проверьте удаление.

**Создание и удаление файлов:**

*   Создайте файл `newfile1` командой `touch newfile1`. Проверьте его создание через `ls -l /tmp/newfile1`.
*   Создайте файл `newfile2`, используя перенаправление вывода: `> newfile2`. Проверьте создание через `ls -l /tmp/newfile2`.
*   Удалите файл `newfile2` командой `rm newfile2`. Проверьте удаление через `ls /tmp/newfile2`.

### 3. Копирование и перемещение файлов в файловой системе

Действия выполняются в каталоге `/tmp`.

*   Перейдите в `/tmp` с помощью `cd /tmp`.

**Копирование файлов:**

*   Скопируйте файл `/etc/fstab` в `/tmp` командой `cp /etc/fstab /tmp`. Проверьте результат через `ls -l /tmp/fstab`. Сравните время создания исходного и скопированного файлов с помощью `ls -l /etc/fstab`. Сохраните вывод.
*   Скопируйте `/etc/passwd` в `/tmp` с сохранением атрибутов: `cp -p /etc/passwd /tmp`. Проверьте результат через `ls -l /tmp/passwd`. Убедитесь, что временные метки совпадают с оригиналом (`ls -l /etc/passwd`).

**Перемещение файлов:**

*   Переместите файл `fstab` в `fstab_new` командой `mv fstab fstab_new`. Проверьте


успешность перемещения через `ls /tmp/fstab /tmp/fstab_new`.

### 4. Копирование и перемещение каталогов в файловой системе

Действия выполняются в каталоге `/tmp`.

*   Перейдите в `/tmp` с помощью `cd /tmp`.

**Копирование каталогов:**

*   Создайте вложенные каталоги `dir_a/dir_b` командой `mkdir -p dir_a/dir_b`. Проверьте результат через `ls -d /tmp/dir_a/dir_b`.
*   Скопируйте каталог `dir_a` вместе с вложенным содержимым в `dir_a_copy1` командой `cp -r dir_a dir_a_copy1`. Проверьте результат через `ls -l /tmp/dir_a_copy1`. Сравните временные метки с оригиналом.
*   Скопируйте `dir_a` в `dir_a_copy2` с сохранением атрибутов: `cp -ra dir_a dir_a_copy2`. Проверьте результат через `ls -l /tmp/dir_a_copy2`. Убедитесь, что временные метки теперь совпадают.

**Перемещение каталогов:**

*   Переместите каталог `dir_a` в `dir_a_new` командой `mv dir_a dir_a_new`. Проверьте результат через `ls dir_a dir_a_new`.

### 5. Использование шаблонов при работе с файлами и каталогами

Действия выполняются в каталоге `/tmp`.

*   Перейдите в `/tmp` с помощью `cd /tmp`.

*   Создайте несколько файлов с похожими именами: `touch file1 file2 file3 file4`.
*   Выведите список файлов, начинающихся с `file`, используя шаблон: `ls -l file*`. Сохраните вывод.
*   Удалите все файлы, начинающиеся с `file`, командой `rm file*`. Проверьте удаление через `ls -l file*`.

### 6. Поиск файлов и каталогов

*   Найдите файл `.bashrc` в домашнем каталоге: `find ~ -name '.bashrc'`. Сохраните вывод.
*   Найдите все файлы с расширением `.png` в домашнем каталоге: `find ~ -name '*.png'`. Сохраните вывод.
*   Найдите каталоги с именем `Pictures`: `find ~ -name 'Pictures'`. Сохраните вывод.

### 7. Создание и использование ссылок на файлы и каталоги

Действия выполняются в каталоге `/tmp`.

*   Перейдите в `/tmp` с помощью `cd /tmp`.

**Создание ссылок:**

*   Создайте символическую ссылку на `/etc/fstab` с именем `fstab_link`: `ln -s /etc/fstab fstab_link`. Проверьте через `ls -l fstab_link`.
*   Создайте ссылку на каталог `/etc` с именем `etc_link`: `ln -s /etc etc_link`. Проверьте через `ls -ld etc_link`.
*   Создайте файл `new_file` и ссылку на него: `ln -s new_file new_file_link`. Проверьте ссылку. Удалите исходный файл (`rm new_file`) и снова выполните `ls -l new_file_link`. Обратите внимание на изменение вывода (ссылка стала "мёртвой").

### 8. Оценка занятого и свободного пространства

*   Определите объём, занимаемый каталогом `/etc`, командой `du -s -h /etc`. Сохраните вывод.
*   Запустите программу `ncdu /usr`, чтобы в удобном виде оценить пространство, занимаемое `/usr`. Сделайте скриншот окна терминала, затем закройте программу клавишей `q`.
*   Получите информацию об общем, используемом и доступном пространстве на дисках: `df -h`. Сохраните вывод.

**Контрольные вопросы**

1. **Какую команду необходимо использовать для вывода текущего каталога?**  
   `pwd`

2. **Какую команду необходимо использовать для вывода содержимого каталога?**  
   `ls`

3. **Какую команду необходимо использовать для смены текущего каталога?**  
   `cd`

4. **Какую команду необходимо использовать для создания нового каталога?**  
   `mkdir`

5. **Какую команду необходимо использовать для удаления каталога?**  
   `rmdir` (для пустого) или `rm -r` (для непустого)

6. **Какую команду необходимо использовать для создания пустого файла?**  
   `touch`

7. **Как обозначается оператор перенаправления вывода?**  
   `>` (перезапись) или `>>` (добавление)

8. **Какую команду необходимо использовать для копирования объектов?**  
   `cp`

9. **Какую команду необходимо использовать для перемещения объектов?**  
   `mv`

10. **Какую команду необходимо использовать для удаления объектов?**  
    `rm`

11. **Какую команду необходимо использовать для поиска объектов?**  
    `find`

12. **Какую команду необходимо использовать для создания символических ссылок?**  
    `ln -s`

13. **Какую команду необходимо использовать для получения занимаемого каталогом дискового пространства?**  
    `du`

14. **Какую команду необходимо использовать для отображения занимаемого каталогом дискового пространства в наглядном виде?**  
    `du -h`

15. **Какую команду необходимо использовать для получения информации об используемом дисковом пространстве?**  
    `df`
